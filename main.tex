\subsubsection{Pregunta 3: Definición y Justificación de Reglas de Negocio}

Para que los datos crudos se transformen en información útil y confiable para la \textbf{UTA}, el sistema de "Préstamos de Equipos" debe regirse por reglas que aseguren la integridad y consistencia. A continuación, se detallan las 5 reglas principales con su respectivo porqué:

\begin{enumerate}
    \item \textbf{Validación de Estado Académico y Financiero}
    \begin{itemize}
        \item \textbf{Regla:} Solo usuarios con \texttt{estado = 'Activo'} y \texttt{saldo\_biblioteca = 0} pueden registrar un nuevo préstamo.
        \item \textbf{Justificación:} Esta regla garantiza la \textbf{solvencia administrativa}. Evita que activos de la universidad sean entregados a personas que no tienen un vínculo vigente o que mantienen deudas, protegiendo el patrimonio institucional.
    \end{itemize}

    \item \textbf{Control de Exclusividad y Concurrencia (Anti-Duplicidad)}
    \begin{itemize}
        \item \textbf{Regla:} Un \texttt{id\_equipo} no puede estar asociado a dos registros de \texttt{Prestamo} activos simultáneamente.
        \item \textbf{Justificación:} Previene el error lógico de "doble asignación". Sin esta regla, el sistema podría reportar que una misma laptop está en dos facultades distintas, destruyendo la veracidad de la información de inventario.
    \end{itemize}

    \item \textbf{Automatización de Sanciones por Temporalidad}
    \begin{itemize}
        \item \textbf{Regla:} Al superar la \texttt{fecha\_devolucion}, el sistema debe cambiar automáticamente el estado del usuario a \texttt{'Moroso'}.
        \item \textbf{Justificación:} Asegura la \textbf{disponibilidad de recursos}. La penalización automática incentiva la devolución puntual, permitiendo que el flujo de equipos no se detenga por negligencia del usuario.
    \end{itemize}

    \item \textbf{Integridad de Identificación Física (Inventario)}
    \begin{itemize}
        \item \textbf{Regla:} No se permite el ingreso de préstamos si el \texttt{id\_etiqueta\_institucional} no existe en la tabla \texttt{Activos\_Fijos}.
        \item \textbf{Justificación:} Evita el ingreso de "datos fantasma". Garantiza que cada equipo prestado esté debidamente codificado y bajo la responsabilidad del departamento de tecnologías de la UTA.
    \end{itemize}

    \item \textbf{Restricción de Capacidad por Bloque Horario}
    \begin{itemize}
        \item \textbf{Regla:} El número de préstamos activos no puede exceder la capacidad física configurada en la entidad \texttt{Aula}.
        \item \textbf{Justificación:} Sirve para la \textbf{planificación logística}. Evita el hacinamiento en laboratorios y permite al sistema sugerir aulas alternativas si una facultad ha alcanzado su límite de ocupación.
    \end{itemize}
\end{enumerate}

\subsection{Conclusiones}
\begin{itemize}
    \item Se concluye que en la \textbf{UTA} los datos (como un número de cédula o una hora de marcación) no sirven de mucho por sí solos; necesitan ser organizados por el sistema para convertirse en información valiosa, como saber si un estudiante tiene derecho a graduarse o si un laboratorio está lleno.
    
    \item Identificar las 8 entidades nos permitió entender cómo organizar el sistema de préstamos. Al separar los datos en grupos claros (Estudiantes, Equipos, Multas, etc.), aseguramos que cada detalle importante esté guardado en su lugar y sea fácil de encontrar sin confusiones.
    
    \item Se determinó que las reglas son esenciales para que el sistema funcione bien en la vida real. Sin ellas, no habría control sobre quién devuelve los equipos a tiempo o quién tiene deudas, lo que causaría desorden y falta de equipos para otros compañeros.
\end{itemize}

\subsection{Recomendaciones}
\begin{itemize}
    \item Se recomienda que el sistema esté conectado siempre a la base de datos central de la universidad para que los nombres y correos de los estudiantes estén siempre actualizados, evitando errores al momento de registrar un préstamo.
    
    \item Es aconsejable que al diseñar las tablas no se repita información innecesaria. Por ejemplo, es mejor guardar la carrera del estudiante una sola vez en su perfil en lugar de anotarla en cada préstamo, para que la base de datos sea más limpia y rápida.
    
    \item Se sugiere que el sistema envíe notificaciones automáticas a los usuarios cuando falte poco para su hora de entrega. Esto ayudaría a que se cumplan las reglas de forma más amigable y se mantenga una mejor rotación de los equipos en el laboratorio.
\end{itemize}
